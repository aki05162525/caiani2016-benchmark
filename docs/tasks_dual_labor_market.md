# Caiani(2016) ベンチマークモデル：二重労働市場（R/N）拡張 実装タスク

## 0. 目的・スコープ

* 目的：Caiani(2016) SFC-ABM の構造を極力維持しつつ、**労働市場のみ**を二重化（正規 R / 非正規 N）する。
* スコープ内（差分の中心）：

    * 企業の労働需要：`N → (N_R, N_N)`、需要分解 `(N^{D,R}, N^{D,N})`
    * 雇用調整：分離・解雇・求人・採用をタイプ別化
    * 労働市場：タイプ別にマッチング（失業プールも二重）
    * 賃金：要求賃金（留保賃金）更新のタイプ別化
    * 失業・ドール：タイプ別失業率、平均賃金の加重計算
    * 生産・価格の整合：CES 実効労働で労働制約を反映、価格式の労働費用を2種類に
* スコープ外（維持）：

    * 財市場配分、価格・マークアップ更新（労働費用項を除く）
    * 投資、信用供給、デフォルト、政府・中銀、銀行会計、SFC恒等式

## 4. 労働需要の分解（CES：中核タスク）

### 4.1 CES 実効労働関数を実装

* [ ] `N_eff(N_R, N_N) = [ δ (A_R N_R)^ρ + (1-δ)(A_N N_N)^ρ ]^(1/ρ)` を関数化
* [ ] 数値安定性：

    * `N_R, N_N ≥ 0`
    * `ε` を使ったゼロ割回避や安全な pow
    * `ρ` が 0 に近い場合の扱い（本仕様では ρ≠0 だが guard は必要）

**Done 定義**：単体テストで単調性（N_R↑でN_eff↑など）と非負性が確認できる。

### 4.2 需要比率式（FOC）を実装

* [ ] 比率 `ratio = N^{D,R} / N^{D,N}` を計算：

    * `ratio_raw = (δ/(1-δ))^(1/(1-ρ)) * (A_R/A_N)^(ρ/(1-ρ)) * (W_N^e/W_R^e)^(1/(1-ρ))`
* [ ] クリップ：`ratio = clip(ratio_raw, φ_min, φ_max)`

**Done 定義**：賃金比が変わると ratio が合理的に動くことを確認（例：W_R^e↑→ratio↓）。

---

### 4.3 `(N^{D,R}, N^{D,N})` の閉形式計算（ratio を所与として配分）

* [ ] **前提**：4.2 で `ratio = N^{D,R} / N^{D,N}` を算出し、`ratio = clip(ratio_raw, φ_min, φ_max)` を適用する  
  ※ clip 後は「FOCの厳密解」ではなくなる可能性があるが、安定性を優先して ratio を所与として配分する。

* [ ] **分母項（scale factor）の計算**：  
  `Denom = [ δ * (A_R * ratio)^ρ + (1-δ) * (A_N)^ρ ]^(1/ρ)`  
  `Denom = max(Denom, ε)`（ゼロ割・数値不安定のガード）

* [ ] **タイプ別需要の確定**：  
  `N^{D,N} = N^D / Denom`  
  `N^{D,R} = ratio * N^{D,N}`

* [ ] **早期リターン**：`N^D < ε` の場合は `N^{D,R}=0, N^{D,N}=0`

**Done 定義**：`N_eff(N^{D,R}, N^{D,N})` を再計算すると、許容誤差内で `N^D` に一致する（例：相対誤差 < tol）。

## 5. 雇用調整（分離・解雇・求人）をタイプ別化

### 5.1 分離

* [ ] `Sep_ℓ = ϑ_ℓ * N_ℓ(t-1)`
* [ ] `tildeN_ℓ = N_ℓ(t-1) - Sep_ℓ`

### 5.2 解雇（硬直性）

* [ ] `Fire_ℓ = η_ℓ * max(0, tildeN_ℓ - N^{D,ℓ})`
* [ ] `hatN_ℓ = tildeN_ℓ - Fire_ℓ`

### 5.3 求人

* [ ] `V_ℓ = max(0, N^{D,ℓ} - hatN_ℓ)`

**Done 定義**：需要が減ったとき、正規の方が解雇されにくい（η_R < η_N）がシミュレーション上で発現する。

---

## 6. 労働市場：タイプ別マッチング（2回回す）

### 6.1 失業プールの二重化

* [ ] `U_ℓ = LF_ℓ - Σ_{x,i} N_{x,i,ℓ}`
* [ ] `u_ℓ = U_ℓ / LF_ℓ`

### 6.2 マッチング（Caiani の探索・採用を踏襲）

* [ ] 各タイプ ℓ について：

    * 企業は `V_{x,i,ℓ}` 件の求人
    * 各求人は失業プールから候補者を `χ_ℓ` 人サンプル
    * 企業は候補者の要求賃金 `w^d` を見て採用（例：最小要求賃金優先）
    * `Hire_{x,i,ℓ}` を確定
* [ ] 採用後更新：`N_ℓ = hatN_ℓ + Hire_ℓ`

**Done 定義**：R 市場の求人が N タイプ家計に割り当てられないことがコード上保証される（混線なし）。

---

## 7. 賃金：要求賃金更新のタイプ別化

### 7.1 要求賃金更新（Caiani式のタイプ別）

* [ ] 直近4期の失業履歴（家計ごとの `u_{h,t-n}`）を参照
* [ ] 条件分岐（仕様どおり）：

    * 失業が多い → 下方調整
    * 失業が少なく、かつ `u_{ℓ,t-1} ≤ υ_ℓ` → 上方調整
    * それ以外 → 据置
* [ ] Folded Normal の生成をタイプ別パラメータで（または倍率で）差をつける

**Done 定義**：同じ景気局面でも、非正規の方が賃金上昇が抑制される設定が可能。

### 7.2 タイプ別平均賃金と期待賃金

* [ ] `W_ℓ,t = mean(w_{h,t} | employed & type=ℓ)`
* [ ] 期待賃金 `W^e_ℓ` を適応的期待で更新

**Done 定義**：価格式・需要分解で参照される `W^e_R, W^e_N` が時系列で一貫して更新される。

---

## 8. ドール（失業給付）と家計消費の最小拡張

### 8.1 平均賃金（加重平均）とドール

* [ ] `bar_w = (Σ_ℓ W_ℓ * N_ℓ_total) / (Σ_ℓ N_ℓ_total)` を実装
* [ ] `dole = ω * bar_w`

**Done 定義**：労働市場の構造変化（R/N比率）でドールが自然に変動する。

### 8.2 消費関数の係数のみタイプ別化

* [ ] `C_h = c1_{type(h)} * YD_h + c2 * D_{h,t-1}`
* [ ] 必要なら `C_h ≤ D_{h,t-1} + YD_h` のクリップ

**Done 定義**：雇用形態の違いが需要側に最低限フィードバックする。

---

## 9. 生産・価格：整合条件の更新

### 9.1 生産（労働不足の安全弁）

* [ ] マッチング後の `N_R, N_N` から `N_eff` を計算
* [ ] 資本財企業：`y_k = min(y^D_k, μ_N * N_eff)`
* [ ] 消費財企業：労働不足時の稼働率調整

    * `u = min(u^D, (N_eff*l_k)/k )` の形で整合させる
    * `y_c = u * k * μ_K`

**Done 定義**：労働不足が起きたときに、生産が矛盾なく下方修正される。

### 9.2 価格式（労働費用を二重化）

* [ ] 単位労働費用の計算を
  `W^e * N^D → W^e_R*N^{D,R} + W^e_N*N^{D,N}` に置換
* [ ] それ以外のマークアップ更新は Caiani のまま

**Done 定義**：価格が「2種類の期待賃金」と「需要分解」に整合して決まる。

## 10. SFC（会計）・集計の差分反映

* [ ] 企業の賃金支払いフローを2種類に分割（ただし部門間符号は不変）
* [ ] 集計統計（出力用指標）にタイプ別系列を追加：

**Done 定義**：従来系列を壊さず、二重化された系列が追加される。

## 11. テスト計画（最低限）

### 11.1 単体テスト

* [ ] `N_eff` の性質（非負、単調、境界）
* [ ] 需要分解ソルバが `N_eff ≈ N^D` を満たす
* [ ] ratio clip が効く（極端な賃金比でも暴れない）

### 11.2 統合テスト（1ステップ）

* [ ] 分離→解雇→求人→採用→雇用更新がタイプ別に一貫
* [ ] R/N の混線（誤採用）が起きない

### 11.3 挙動確認（簡易）

* [ ] `η_R < η_N` で、ショック時に R が保持されやすい
* [ ] `υ_N < υ_R` で、好況時も N の要求賃金が上がりにくい

## 12. 実装順序（推奨ロードマップ）

1. **データ構造の二重化（企業 N / 家計 type）**
2. **CES 実効労働 + 需要分解ソルバ**
3. **雇用調整（Sep/Fire/Vacancy）をタイプ別化**
4. **労働市場マッチングをタイプ別に2回**
5. **要求賃金更新と W / W^e の整合**
6. **ドール・消費のタイプ別化（最小）**
7. **生産・価格の整合更新**
8. **集計指標 + テスト + 出力確認**
